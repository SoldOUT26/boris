ОСНОВА

Программа закрытая: нет API, веб-интерфейса и выгрузки. Поэтому агент работает как ассистент врача — открывает вкладки, читает экран, жмёт кнопки и формирует заключения. Сервер с локальной моделью за 1–1,5 млн ₽ не нужен: используем коммерческие модели c zero-data-retention (SOC 2) — данные не уходят на обучение, ядро недоступно.

CO-BUILDING

Главная прикладная часть. Всю сложную инженерную работу — архитектуру агента, способ «видеть» закрытое ПО и настройку workflow — мы берём на себя. Поскольку у программы нет API и выгрузки, агент управляет интерфейсом одним из двух способов:

🌳 Accessibility-дерево Windows · UI Automation
Точный машинный доступ к элементам интерфейса, кнопкам и полям. Быстро, стабильно, не зависит от разрешения и положения окна.
👁 Визуальный ИИ-агент · Vision API
Агент распознаёт экран и действует как врач-оператор. Работает там, где интерфейс закрыт для машинного доступа.

Выбираем оптимальный вариант под вашу программу или комбинируем оба. На этой основе строим рабочий поток: агент открывает программу, заводит пациента, грузит файлы, ведёт анализ по вашим критериям и формирует заключение. Ваши знания врача переносятся в инструкции агента — он повторяет логику на потоке. После сборки мы передаём все исходные данные и материалы и обучаем тому, как это модернизировать, улучшать, обновлять и дообучать.
